Tổng Quan Hệ Thống

Mục tiêu của dự án là xây một hệ sinh thái du lịch ẩm thực Quận 4 gồm: website công khai cho du khách, trang admin để vận hành, luồng owner để đối tác đề xuất/cập nhật nội dung, mobile app để trải nghiệm di động, và backend trung tâm làm API + lưu dữ liệu + analytics + media + audio.

Ứng dụng 1

Website User

Khách xem POI, khám phá, quét QR, tìm gần tôi, xem bản đồ, xem tour, nghe audio, đăng ký tài khoản.

Ứng dụng 2

Website Admin

Admin quản lý category, POI, users, owner registrations, submissions, audio, localization, analytics, QR, tour, map pack.

Ứng dụng 3

Backend API

Xử lý auth, CRUD, upload media, audio generation, QR activation, analytics, owner workflow, map pack và health check.

Ứng dụng 4

Mobile App

Khám phá trên di động với cache offline, audio offline, geofence, map fallback, QR / POI code flow.

Tư duy đúng về dự án: đây không chỉ là một website hiển thị dữ liệu. Thực chất nó là một hệ thống nội dung du lịch ẩm thực có vận hành nội bộ, có đối tác owner, có phân phối nội dung qua QR, audio, map và có analytics theo hành vi người dùng.

Kiến Trúc Và Thành Phần

1. Client layer

  • website-user: React 19, Vite, Tailwind, TanStack Query, Zustand, Framer Motion, MapLibre, PWA.
  • website-admin: React 19, Vite, Ant Design, Zustand, TanStack Query, Recharts.
  • mobile/Quan4CulinaryTourism.Mobile: .NET MAUI, SQLite local cache, audio plugin, MAUI Maps.

2. Server layer

  • backend/Quan4CulinaryTourism.Api: ASP.NET Core Web API.
  • MongoDB lưu document cho users, POI, analytics, owner registrations, owner submissions, QR activations, tours, audio, localizations.
  • wwwroot/uploads dùng cho media/audio file local.

Luồng tổng quát

Bước 1: Admin seed / tạo dữ liệu nền như category, POI, QR, tours, audio, localization.
Bước 2: Public web và mobile gọi API để tải category, POI, audio, QR activation, tours.
Bước 3: Người dùng tương tác, backend nhận analytics events như mở POI, nghe audio, tìm kiếm, nearby, presence ping.
Bước 4: Admin xem dashboard, analytics, usage history và điều hành nội dung dựa trên dữ liệu thật.
Bước 5: Owner có thể đăng ký, được duyệt, rồi gửi submission tạo mới hoặc cập nhật POI để admin duyệt.

Khám phá nội dung

Nghe thuyết minh

Audio player phát file audio hoặc dùng TTS fallback khi không có file.

Tìm gần tôi

Xem đường đi trên map

Map page lấy vị trí thật, chọn POI, vẽ route, hiển thị quãng đường và thời gian.

Quét QR / nhập mã

Mở nhanh nội dung POI qua QR activation hoặc mã POI fallback.

Đăng ký owner

Từ trang account, user có thể gửi form xin trở thành owner.

Luồng quan trọng trên public web

Luồng xem POI: vào /explore → chọn POI → mở /poi/:id → nghe audio / xem giờ mở cửa / bấm chỉ đường.
Luồng nearby: vào /nearby → cấp GPS hoặc dùng vị trí Quận 4 → backend gọi endpoint nearby → render card POI + distance.
Luồng bản đồ: vào /map hoặc từ chi tiết POI → lấy current location → chọn POI → gọi route service → render route line trên MapLibre.
Luồng QR: người dùng quét / nhập mã → FE resolve sang activation hoặc POI → chuyển thẳng đến nội dung đúng → ghi analytics qr_scanned.

Luồng Owner

Owner là role trung gian giữa khách công khai và admin vận hành. Ý tưởng là đối tác có thể đóng góp nội dung nhưng vẫn phải đi qua bước duyệt của admin.

Luồng 1: user xin làm owner

1. User đăng ký / đăng nhập vào public web.
2. Vào /account và gửi form owner registration.
3. Backend tạo OwnerRegistration ở trạng thái pending.
4. Admin vào /admin/owner-registrations để approve hoặc reject.
5. Nếu approve, user được gắn role Owner và mở được /owner.

Luồng 2: owner gửi submission

1. Owner vào workspace /owner.
2. Có thể gửi submission kiểu create hoặc update.
3. Submission lưu đầy đủ thông tin POI đề xuất.
4. Admin vào /admin/submissions để duyệt.
5. Nếu duyệt create, backend tạo POI mới. Nếu duyệt update, backend áp thay đổi lên POI cũ.
Luồng owner được thiết kế để tránh việc đối tác sửa dữ liệu production trực tiếp. Họ chỉ gửi đề xuất; quyền quyết định cuối cùng vẫn là admin.

Backend API

Backend là lõi nghiệp vụ. Từ góc nhìn kiến trúc, backend chia thành: Controllers → Services → Repositories → MongoDB / file storage.

Nhóm controller hiện có

Auth / User

AuthController, AdminController, OwnerController

Nội dung

CategoryController, PoiController, LocalizationController, AudioController, MediaController

Phân phối / theo dõi

QrActivationController, TourController, MapsController, AnalyticsController, HealthController

Nghiệp vụ backend nổi bật

POI + nearby + search

  • Load all public POI.
  • Search theo keyword / category / price.
  • Nearby theo tọa độ và bán kính.
  • Map POI sang response có ảnh, tags, rating, location.

Audio generation

  • FE gọi audio endpoint.
  • Nếu chưa có audio, backend có thể tự generate MP3 bằng Python + gTTS.
  • File được lưu ở uploads và metadata ghi vào Mongo để tái sử dụng.

Analytics

  • Thu event từ web/mobile.
  • Tính summary, top views, top audio, heatmap, route traces.
  • Tính active visitors realtime từ heartbeat presence_ping.

Owner moderation

  • Approve/reject owner registrations.
  • Approve/reject submissions.
  • Áp submission vào dữ liệu POI thật.

Ứng Dụng Mobile

Mobile app là bề mặt trải nghiệm giàu tính di động hơn website, tập trung vào offline usage, audio, QR và geofence. Nó không chỉ là bản sao của web.

Offline cache

Lưu categories, POI list, POI detail và audio metadata/local path vào SQLite.

Offline audio

Tải audio về FileSystem.AppDataDirectory/audio để mở lại khi mất mạng.

Geofence demo

Tự động narration cơ bản khi vào vùng POI, có cooldown để tránh spam.

QR / mã POI

Có màn demo scan hoặc nhập mã POI / QR flow.

Map fallback

Nếu chưa cấu hình Google Maps key, app fallback sang danh sách POI và nút mở Google Maps ngoài app.

Base URL linh hoạt

Cho phép chọn preset hoặc nhập tay backend base URL trong settings.

Các Luồng Cắt Ngang Quan Trọng

1. Luồng audio

Website/mobile gọi API lấy audio theo poiId + lang.
Backend kiểm tra đã có PoiAudio hay chưa.
Nếu có: trả URL file audio.
Nếu chưa có: có thể auto generate từ ttsScript hoặc description, lưu file và lưu metadata.

2. Luồng bản đồ và chỉ đường

Public web lấy GPS thật từ browser.
PoiMap render marker user + marker POI bằng MapLibre.
Route service gọi OSRM để lấy geometry, distance, duration.
UI hiển thị route line, số km, thời gian và danh sách điểm bên cạnh.

3. Luồng QR activation

Admin tạo QR activation gắn với POI / stop zone / scan mode.
Khách quét QR trên public web hoặc mobile.
Frontend resolve mã QR → activation → POI target.
Hệ thống mở nội dung đúng, có thể ưu tiên audio / TTS tùy scan mode.

4. Luồng analytics và online realtime

Event tracking: public web/mobile gửi các sự kiện như poi_viewed, audio_played, search_executed, qr_scanned.
Heartbeat: public web còn gửi presence_ping khi tab đang visible.
Backend tổng hợp heartbeat trong cửa sổ ngắn để tính khách đang online hiện tại, kể cả ẩn danh.
Admin xem realtime ở dashboard và analytics page theo polling định kỳ.

Nên Đọc Repo Theo Thứ Tự Nào

  1. Đọc repo root README.md để biết 4 ứng dụng và cách chạy.
  2. Đọc website-user/src/App.tsx để hiểu public routes và luồng người dùng.
  3. Đọc website-admin/src/routes/AppRoutes.tsx để hiểu toàn bộ bề mặt quản trị.
  4. Đọc các controller trong backend/.../src/Controllers để thấy các nhóm API.
  5. Đọc PoiService, AdminService, AnalyticsService, OwnerService để hiểu nghiệp vụ lõi.
  6. Đọc mobile README và các page/viewmodel quan trọng nếu muốn hiểu offline/geofence/audio trên di động.
File này được viết để giúp nắm toàn bộ dự án theo góc nhìn nghiệp vụ + kiến trúc + luồng sử dụng, không thay thế tài liệu API chi tiết. Khi cần đào sâu hơn, nên mở thêm: docs/API_CONTRACT.md và docs/API_TEST_FLOW.md.
